# PatterKnitter

# Knitting Chart Generator - Domain Definitions

## Purpose

This application generates colourwork charts and optional written instructions for stocking stitch knitting.

The application does not generate complete knitting patterns. Users are expected to incorporate generated charts and instructions into their own knitting projects and patterns.

---

## Design

A visual motif, image or piece of artwork created by the generator.

A design contains no knitting instructions and represents only the intended appearance of the finished knitted fabric.

Examples:

* Heart motif
* Snowflake motif
* Checkerboard
* Pixel-art animal

---

## Chart

A knitting chart generated from a design.

A chart is a grid where:

* Each column represents one stitch.
* Each row represents one knitted row.
* Each cell represents one stitch in one row.
* Cells indicate colour only.
* Charts assume stocking stitch throughout.

Charts contain no shaping information, stitch variations, increases, decreases, cables or lace instructions.

---

## Written Instructions

An optional row-by-row textual representation of a chart.

Written instructions provide the same colour information as the chart in a format that can be read without referring to the chart grid.

Examples:

* Row 1: Colour A 6 sts, Colour B 4 sts, Colour A 6 sts
* Row 2: Colour A 4 sts, Colour B 8 sts, Colour A 4 sts

Written instructions describe colour placement only and do not include shaping or stitch pattern instructions.

---

## Chart Pattern

A complete chart output generated by the application.

A chart pattern consists of:

* A design
* A chart
* Optional written instructions

A chart pattern represents a section of knitted fabric and is intended to be incorporated into a larger knitting project or pattern.

---

## Symbol

A character or shape displayed within a chart cell to indicate which colour yarn should be used.

Current implementation supports three colours:

| Symbol | Colour   |
| ------ | -------- |
| Blank  | Colour A |
| X      | Colour B |
| O      | Colour C |

Additional symbols and colours may be supported in future versions.

---

## Chart Section

A rectangular area of knitted fabric represented by a chart pattern.

A chart section may be knitted independently or incorporated into a larger knitting project.

Examples:

* Blanket panel
* Jumper front motif
* Cushion cover panel
* Scarf insert

---

## Project Assumptions

### Terminology

* UK knitting terminology is used throughout.

### Yarn

* Material: Wool
* Weight: Double Knit (DK)

### Needles

* Size: 4 mm
* Type: Single-pointed straight needles

### Stitch Pattern

* Stocking stitch only.
* No alternative stitch patterns are supported.
* Charts represent colour changes only.
* Every chart cell represents one stocking stitch.

### Gauge

All size calculations assume:

```json
{
  "gauge": {
    "stitches_per_10cm": 20,
    "rows_per_10cm": 28
  }
}
```

Derived dimensions:

* 1 stitch = 5.0 mm wide
* 1 row = 3.57 mm high
* Stitch aspect ratio ≈ 1.4 : 1

### Size Calculation

Physical dimensions are estimated using the standard gauge.

Width (cm) = stitches ÷ 20 × 10

Height (cm) = rows ÷ 28 × 10

Examples:

* 20 stitches × 28 rows ≈ 10 cm × 10 cm
* 40 stitches × 56 rows ≈ 20 cm × 20 cm
* 60 stitches × 84 rows ≈ 30 cm × 30 cm

### User Customisation

Users may override the default gauge settings to match their own knitting tension, yarn choice or needle size.

Changing gauge settings affects only the estimated physical dimensions of the finished knitted fabric and does not alter the generated chart pattern.

## Archetype-Based Procedural Pattern Generator

Pattknitter is a browser-based procedural pattern generation system that enables users to design, modify, and export visual patterns through a unified, human-centric parameter model.

Instead of exposing generator-specific parameters, PattKnitter abstracts procedural systems (noise, tessellation, wave fields, spatial patterns, and recursive structures) into a shared set of archetypes such as Size, Density, Detail, and Randomness. These archetypes provide a consistent interface across heterogeneous pattern generation algorithms.

Patterns are defined as normalized recipes consisting of:

a generator (e.g. noise, grid, Voronoi)
a set of parameter-to-archetype mappings
transform rules for consistent UI scaling
export capabilities

The system supports real-time preview and export to common formats including PNG and SVG, with an experimental pathway for domain-specific export into knitting pattern grids.

Key Features
Archetype-driven parameter system for procedural patterns
Multiple generator families (tessellation, noise, wave, recursive, spatial)
Consistent UI across all pattern types
Live canvas-based preview (p5.js)
Export to SVG / PNG
Experimental knitting chart export (grid-based patterns)
Research Focus

This project explores whether a normalized archetype-based interaction model can provide a consistent and usable interface for heterogeneous procedural pattern systems, evaluated through user testing with domain-specific (textile/knitting) workflows.
